# TechShop

## Практика 7 (JWT-аутентификация)
- Реализована регистрация и вход пользователя с выдачей `accessToken` и `refreshToken`.
- Добавлен защищенный маршрут получения текущего пользователя `GET /api/auth/me`.
- На фронтенде настроено сохранение токенов и автоматическая авторизация после входа.

## Практика 8 (Refresh Token)
- Реализовано обновление пары токенов через маршрут `POST /api/auth/refresh`.
- На клиенте добавлен механизм автоматического обновления access-токена при ответе `401`.
- Сессия пользователя сохраняется между перезагрузками страницы.

## Практика 9 (Защищенные маршруты и API)
- Backend и frontend объединены в одно приложение интернет-магазина `Tech Shop`.
- Реализованы маршруты для работы с товарами: создание, получение списка, получение по `id`, обновление и удаление.
- Добавлены middleware для логирования запросов, обработки `404` и серверных ошибок.

## Практика 10 (Авторизация и интерфейс)
- Создан интерфейс входа и регистрации на React.
- Пользователь после входа получает доступ к рабочему интерфейсу приложения.
- Реализован удобный frontend для просмотра и управления товарами через REST API.

## Практика 11 (RBAC)
- Добавлена система ролей: `Гость`, `Пользователь`, `Продавец`, `Администратор`.
- Пользователь может только просматривать товары.
- Продавец может создавать и редактировать товары.
- Администратор может управлять товарами и пользователями.
- Реализованы маршруты `/api/users` для просмотра, редактирования и блокировки пользователей.
- Добавлен `roleMiddleware` для ограничения доступа к маршрутам по ролям.

## Что реализовано в проекте
- Backend на **Node.js + Express**
- Frontend на **React**
- JWT-аутентификация и refresh token
- RBAC-модель доступа
- CRUD для товаров
- CRUD-операции администратора для пользователей
- Swagger-документация для backend
- Обработка ошибок и защита маршрутов

## Структура проекта
- `backend` — серверная часть приложения
- `frontend` — клиентская часть приложения

## Запуск проекта

### Backend
1. Перейти в папку `backend`
2. Установить зависимости:

```bash
npm install
```

3. Запустить сервер:

```bash
node app.js
```

Backend будет доступен по адресу:

`http://localhost:3000`

Swagger-документация:

`http://localhost:3000/api-docs`

### Frontend
1. Перейти в папку `frontend`
2. Установить зависимости:

```bash
npm install
```

3. Запустить приложение:

```bash
npm start
```

Frontend будет доступен по адресу:

`http://localhost:3001`
